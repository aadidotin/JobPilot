## The A+B Combination (headline decision)

Step 0C-bis offered: **A)** execute the locked engineering plan as-is (discovery → application only), **B)** bulk-adopt a post-send layer (follow-ups, referrals, prep) sight unseen, or **C)** productize for multiple users (rejected on premises). The user asked whether A and B could combine. Answer: yes — **A as untouchable baseline + B's content evaluated per-item in the cherry-pick ceremony below.** The combination beat B-alone because bulk adoption would have accepted the interview prep pack (correctly deferred — zero value until an interview exists) without scrutiny, and beat A-alone because four of the five accepted riders cost minutes each on infrastructure the baseline already builds.

## Vision

### 10x Check
JobPilot as scoped ends at "application sent." The 10x version is a **job-search operating system**: discovery → application → follow-up → referral → interview prep → offer, all through one Telegram bot and one SQLite file. The platform insight: the `applications` table + outcome tracking is infrastructure — every post-send feature is a cheap rider on the data spine the baseline already builds. The user stops running a job search and starts operating one: the system remembers every application, chases every silence, surfaces every warm path, and converts every scored rejection into a learning roadmap.
